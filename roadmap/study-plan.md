# Study Plan

## Operating model

Use quiz-driven learning with official documentation as the source of truth.

```text
quiz -> identify weak service/concept -> read official docs -> update note -> re-quiz
```

## Daily loop

1. Answer 5-10 questions.
2. Mark every uncertain answer, not only wrong answers.
3. Convert each gap into a service note, concept note, or comparison note.
4. Add official references.
5. Re-test weak topics after 2-7 days.

## Weekly loop

1. Pick one exam target.
2. Read the official exam guide.
3. Map the in-scope services to `official/service-index.md`.
4. Focus on highest-risk domains first.
5. Book or adjust the next exam date.

## Quality bar for notes

A note is useful only when it answers:

- Why does this service exist?
- When is it the best answer?
- When is it the wrong answer?
- What service is it confused with?
- What exact exam wording points to it?
- Which official AWS pages support the claims?

## Forbidden content

Do not add:

- company/customer architecture
- credentials or account IDs
- internal incident details
- copied proprietary training material
- unsupported claims from random blogs

## Preferred source order

1. AWS documentation
2. AWS exam guides
3. AWS Well-Architected Framework
4. AWS Prescriptive Guidance
5. AWS official blogs
6. Secondary materials only as non-authoritative study aids
