# Exam Order

Goal: fastest practical route to Golden Jacket while keeping failure risk controlled.

## Decision principles

1. Use the current AWS official certification lineup, not a historic fixed meaning of "12 crowns".
2. Verify retirement dates and registration availability before committing to an exam.
3. Start with exams that validate existing practical experience and build momentum.
4. Use high-leverage exams to expand AWS service vocabulary.
5. Keep enough spacing for the retake policy if a failure happens.

## Why not start with Cloud Practitioner?

Given the current profile, Cloud Practitioner is probably not the limiting factor. It can be used as a confidence/momentum exam, but it should not block higher-value exams unless there is a tactical reason.

## Why SAA first?

SAA is the best calibration point:

- It overlaps with current AWS operations experience.
- It validates architecture reasoning.
- It exposes gaps in AWS-specific vocabulary.
- It is less risky than starting with a Professional exam.

## Why SAP relatively early?

SAP is high leverage. Its study scope forces coverage of:

- multi-account design
- hybrid networking
- migration patterns
- DR and high availability
- cost and performance optimization
- governance and security tradeoffs

This can make later exams easier.
