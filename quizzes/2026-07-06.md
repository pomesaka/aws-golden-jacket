# Quiz Log: 2026-07-06

Initial diagnostic quiz for AWS Golden Jacket planning.

## Q1: EC2 Auto Scaling and uploaded images

Scenario: Web application stores uploaded images on local EC2 disks. Auto Scaling causes some instances to miss uploaded images.

Answer given: B, EFS.

Correct answer: C, S3.

Learning:

- User-uploaded images are usually object data.
- Default AWS best-practice direction: store objects in S3 and keep EC2 stateless.
- EFS is valid when the app requires POSIX/NFS file-system semantics or minimal code change from local file access.

- `services/s3-vs-efs.md`

## Q2: RDS high read load

Scenario: RDS MySQL CPU high. 95% SELECT, 5% writes.

Answer given: C, Read Replica.

Correct answer: C.

Learning:

- Multi-AZ is for high availability and failover.
- Read Replica is for read scaling.
- Standby DB in normal RDS Multi-AZ is not the read-scaling target.

Service notes:

- `services/rds-replication.md`

## Q3: S3, KMS, private EC2 access

Scenario: EC2 in private subnet must access S3 without internet path. Data encrypted with customer-managed KMS key.

Answer given: B, Gateway VPC Endpoint + IAM role + KMS key policy + bucket policy restricting endpoint.

Correct answer: B.

Learning:

- S3 Gateway VPC Endpoint is the default low-cost private path for S3.
- Interface Endpoints use ENIs and generally have hourly/data processing charges.
- KMS authorization is a separate layer. S3 permission alone is not enough for SSE-KMS encrypted object access.

Service notes:

- `services/kms.md`

## Q4: Global read performance for RDS PostgreSQL

Scenario: Writes only in Tokyo, reads globally, minimal application change, keep availability.

Answer given: B, cross-region Read Replicas.

Correct answer: B.

Learning:

- Aurora Global Database is powerful, but migration is a larger change.
- With existing RDS PostgreSQL and minimal change, cross-region read replicas are the better answer.

Service notes:

- `services/rds-replication.md`

## Q5: Global performance and regional failover for HTTP/HTTPS service

Scenario: Tokyo ALB + EC2. Overseas users are slow. Need automatic regional failover, HTTP/HTTPS only, no app changes, simple operations.

Answer given: did not know Global Accelerator; eliminated C and D; considered CloudFront weaker for server-side dynamic app.

Correct answer: B, AWS Global Accelerator.

Learning:

- CloudFront is a CDN and excels at caching static/dynamic content at edge locations.
- Global Accelerator optimizes traffic routing through the AWS global network and supports fast endpoint failover.
- This was a service-vocabulary gap rather than architecture reasoning failure.

Service notes:

- `services/global-accelerator.md`
